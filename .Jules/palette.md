## 2024-05-19 - ARIA labels for icon-only social links and mobile menu toggles
**Learning:** In a template-generated site lacking a build step, maintaining basic accessibility like ARIA attributes for structural elements (e.g., social footer links and mobile nav) requires patching numerous static HTML files. For icon-only links specifically using FontAwesome, screen readers read the icon name or just "link" unless the anchor tag explicitly specifies an `aria-label` and the nested `<i>` tag is hidden via `aria-hidden="true"`.
**Action:** When adding or modifying global UI components containing icon-only elements, strictly enforce the pattern `<a href="..." aria-label="Description"><i class="..." aria-hidden="true"></i></a>`. Future generator scripts should explicitly include these accessibility attributes in their layout extraction blocks to prevent manual drift.
