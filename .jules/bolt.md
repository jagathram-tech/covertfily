## 2024-05-18 - Prevent Layout Thrashing in Dynamic Search UIs
**Learning:** In a codebase without a virtual DOM (like React or Vue), frequent search inputs on large lists of DOM elements (e.g., tools, formats) causes unnecessary computational overhead if `.textContent` and `.querySelectorAll` are executed on every keystroke. Reading properties dynamically inside a fast event loop is a common anti-pattern, although unlike `.innerText`, `.textContent` does not trigger layout thrashing itself.
**Action:** Always pre-calculate and cache static DOM references and their string representations into a JavaScript array upfront. Iterate over the JS array to determine visibility, then only touch the `.style.display` of matching elements. If lists can be dynamically altered (like source/target format dropdowns), use robust cache invalidation checks such as comparing the reference of the first DOM node instead of relying just on element count.
